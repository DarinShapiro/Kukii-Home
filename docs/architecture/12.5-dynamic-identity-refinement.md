# 12.5 — Dynamic Identity Refinement & Multi-Camera Fusion

**Purpose:** How strategic camera placement with overlapping views enables continuous identity confidence improvement, geometric verification, and retroactive re-evaluation across time and space.
**Status:** drafting

---

## Core principle: Identity is not a moment; it's a pattern across space and time

Single-camera identity is uncertain. Multi-camera identity with overlap is confident.

```
Single camera (doorbell straight-on):
  Person detected, face 0.78 match to "Bob"
  → Confidence: 0.78 (what if it's someone who looks like Bob?)
  → No geometric verification possible
  → Gait invisible (head-on)

Multi-camera (doorbell + driveway side angle):
  Doorbell: face 0.78 match to Bob, poor gait visibility
  Driveway: face 0.52 (poor angle), gait 0.87 match to Bob, timing consistent
  Geometric: person left doorbell at 10:15, appeared in driveway at 10:17
             (spatial plausibility check: can walk 12m in 2min? yes, consistent)
  Temporal: both detections within 2min window
  Fusion: face + gait + geometric + temporal → confidence 0.92 (Bob)
```

The system learns identity through **overlapping views** that expose different biometric signals and allow geometric verification.

## Camera placement strategy for identity

Camera placement should be designed around identity-critical moments, not just broad coverage.

### Identity-critical areas (high-value overlap)

These areas should have **2+ overlapping cameras** with complementary angles:

```
Priority 1 (must have overlap):
  Entry points:
    - Front door: main straight-on + side approach angle
    - Garage entry: front + side
    - Patio door (interior/exterior): interior angle + exterior angle
  
  Why: Entry is peak identity moment. People are front-facing, pause/linger.
       Side angle captures gait. Overlap = face + gait + behavior.

Priority 2 (should have overlap):
  High-traffic internal:
    - Main hallway: camera A + camera B at different points
    - Living room: multiple angles
  
  Why: Residents move through frequently. Overlapping views help distinguish
       residents from guests (behavioral patterns, gait, clothing consistency).

Priority 3 (conditional overlap):
  Life-safety areas:
    - Pool: front + side (for stereo drowning detection)
    - Stairs: top + bottom (for fall detection, stereo verification)
  
  Why: Stereo depth + multi-angle behavioral analysis essential for safety.

Priority 4 (optional overlap):
  Perimeter:
    - Driveway: main + side (vehicle + person identity)
    - Backyard: front + back (if large area)
  
  Why: Reduces false identities (visitor vs. resident), vehicle identification.
```

### Overlap geometry: complementary angles

Overlapping cameras should provide **different informational views**, not just redundancy:

```
Bad overlap (wasted):
  Two cameras 0.5m apart, same angle, same distance to subject
  → Both see face equally; both see gait equally
  → Geometric redundancy, no new information
  → Might as well use one camera

Good overlap (complementary):
  Camera A (doorbell, head-on):
    - Sees: face (clear), mouth movement, eye contact, micro-expressions
    - Misses: full body, gait, posture, clothing on lower body
    
  Camera B (side approach, 3m away):
    - Sees: full body, gait, posture, shoulder alignment, stride length
    - Misses: face detail, micro-expressions
    - Sees: clothing (full outfit)
  
  Fusion: face (from A) + gait (from B) + posture (from B) + clothing (from B)
          = rich multi-modal identity signal
```

### Distance and angle strategy

```
Entry point example:
  
  Camera A (doorbell): 
    Position: on door frame, 1.5m from typical visitor position
    Angle: straight-on (0°), horizontal
    Coverage: faces (excellent), hands/objects (good), gait (poor)
  
  Camera B (side entry approach):
    Position: corner of porch, ~3m from entry path
    Angle: 45–60° to entry path
    Coverage: full body (excellent), gait (excellent), face (medium)
    
  Coverage overlap: 2–3 seconds of shared field (person approaching door)
  
  During overlap window:
    → Can triangulate head position (stereo)
    → Can verify face from two angles
    → Can measure gait stride length
    → Can verify person is same individual (not teleport, not replacement)
```

## Multi-view identity fusion

### Same-person verification across cameras

When a person is detected in two overlapping cameras within a time window:

```
Detection A (doorbell_main, 10:15:32):
  Face: "Bob" match 0.78
  Confidence alone: 0.78
  
Detection B (entry_approach, 10:15:34):
  Face: "Bob" match 0.52 (side angle, poor quality)
  Gait: matches Bob's known gait 0.87
  Confidence alone: 0.70 (better than face alone)
  
Multi-view fusion:
  1. Temporal check: 2 seconds apart, consistent with walking pace (0.6–1.5 m/s)
  2. Geometric check: B position is 3m away from A direction, walkable in 2s
  3. Face fusion: A's face (0.78) + B's gait (0.87) = compound confidence 0.91
  4. Behavioral: posture + clothing visible in B confirms identity
  
  Result: "Bob" with confidence 0.91 (much stronger than either camera alone)
```

### Stereo face verification (calibrated overlap)

When two cameras with calibration have stereo baseline and synchronized frames:

```
Synchronized frames at t=10:15:32:
  Camera A (doorbell): face bounding box [100, 200, 300, 400]
  Camera B (side): face bounding box [450, 250, 550, 350]
  
Calibration (from §14):
  Camera A position: (0, 0, 1.5m)
  Camera B position: (2m, 1.5m, 1.5m)
  
Stereo triangulation:
  3D face position: (0.8m, -0.2m, 1.7m) ← person's head in world frame
  
Stereo face matching:
  Can verify face model is geometrically consistent
  (nose bridge at (0.8, -0.2, 1.7), eyes at expected offsets)
  
  Adds constraint: face geometry from two angles must be consistent
  → False matches (AI generation, deepfakes) often fail this test
  → Significantly boosts confidence
```

## Geometric consistency checking

### Spatial plausibility verification

Before accepting identity across cameras, verify the spatial path makes sense:

```
Scenario: Person detected at camera A (front_door), then camera B (backyard)
Time delta: 20 seconds

Adjacency graph (§13):
  front_door → entryway (3s typical)
  entryway → living_room (2s typical)
  living_room → backyard (8s typical)
  Total: ~13 seconds

Observation: 20 seconds elapsed, 13 seconds needed minimum
Plausibility: ✓ Yes (extra 7s accounts for stopping, looking around)

But if time delta was 2 seconds:
Plausibility: ✗ No (physically impossible to reach backyard in 2s from front)
→ Either different person, or teleport (anomaly detection)
```

### Motion velocity verification

Detect anomalies in movement speed:

```
Detection 1: person at position (0, 0) at time 10:15:00
Detection 2: person at position (50m, 0) at time 10:15:10 (10 seconds later)

Implied velocity: 50m / 10s = 5 m/s = 18 km/h

Check:
  Human walking: 1–1.5 m/s
  Human running: 2–4 m/s
  Bicycle: 5–10 m/s
  Car: > 10 m/s

Observed: 5 m/s → plausible (fast running or bike, possible but unusual)
If observed: 20 m/s → implausible for person (car-speed), likely different person
```

## Temporal evidence accumulation across overlapping views

### Evidence compounds with consistent observations

```
Day 1:
  10:15 — doorbell: "Bob" 0.78 face only
  Confidence: 0.78

Day 1:
  14:30 — driveway (overlapping): "Bob" 0.52 face + 0.87 gait, geometric consistent
  10:15 detection updated: (0.78 + 0.87) / 2 ≈ 0.83
  Confidence: 0.83 (gait consistency retroactively boosts earlier detection)

Day 2:
  09:00 — doorbell + entry_approach overlap: "Bob" 0.91 multi-camera
  All prior Bob detections updated: (0.83 + 0.91) / 2 ≈ 0.87
  Temporal pattern: "Bob visits every afternoon around 14:00"
  New Bob detection at time != 14:00 → confidence reduced (timing anomaly)

Day 3:
  22:00 — doorbell: "Bob" 0.45 face + gait pattern mismatch -0.20
  Cross-check against behavioral profile: Bob never visits at night
  Hypothesis: false positive or different person
  Confidence: 0.45 (timing anomaly + gait mismatch)
  DO NOT retroactively boost past Bob detections
```

## Retroactive re-evaluation with overlap information

### User labels a person: propagate across all cameras

```
Tuesday 14:30 — User: "That person at doorbell is Sarah (friend)"

System immediately:
  1. Create Sarah gallery entry
  2. Extract multi-camera data: doorbell + entry_approach overlapping views
  3. Learn: Sarah's face (doorbell) + gait (entry_approach) signature
  4. Query all past detections (30 days):
     - "Unknown person" detections that might be Sarah
     - "Uncertain identity" detections
  
  5. Re-evaluate each with new Sarah baseline:
     Week 1, day 3 @14:15: doorbell "unknown" 0.35
       → Re-evaluate against Sarah: face 0.78, and does gait match?
       → Check driveway overlap window → yes, gait 0.81 match to Sarah
       → Retroactive confidence: 0.80 (high; now identified as Sarah)
     
     Week 2, day 9 @13:50: driveway "unknown" 0.45
       → Re-evaluate: face 0.52, gait 0.79 match to Sarah
       → Geometric plausibility with doorbell camera? Check overlap zone
       → If yes: retroactive 0.75; if no: keep 0.45 (different person)
  
  6. Update all 23 past Sarah detections with new confidence
  7. Build Sarah behavioral profile from all overlapping views:
     - Typical arrival: Thursday afternoons
     - Gait: distinctive long stride 0.85m, pace 1.3 m/s
     - Clothing: prefers jackets, winter warmer
     - Greeting: direct eye contact, quick hello
```

### Conflict detection: is this really the same person?

```
Day 1: Person at doorbell 10:00 (face 0.80 match to "Bob")
Day 1: Person at driveway 10:02 (gait 0.82 match to "Bob")
Both treated as Bob (0.80 confidence)

Day 5: Same person re-detected at doorbell 14:00
But facial hair is gone, gait is slightly different (posture change?)
Confidence: 0.75 (slightly lower, but still Bob)

Day 10: Very similar person at doorbell 09:00
Face: 0.78 match to Bob
Gait: mismatch from earlier Bob observations (0.55)
Entry approach camera: gait is 0.25 mismatch to Bob (someone else!)
Geometric check: person took different entry path (south vs. north normally)

Hypothesis conflict:
  Could be Bob with different posture (injury, backpack weight)
  OR different person who looks like Bob
  
System response:
  Primary: Bob (0.60) — face match, but gait inconsistency
  Alternative: "Bob_lookalike" (0.35) or new visitor (0.35)
  Action: flag for user confirmation (show montage of doorbell vs. earlier Bob)
  User: "That's Bob's brother, never seen him before"
  Result: create new actor "Robert_brother", update all 3 day-10 detections
```

## Per-camera preprocessing optimization across overlapping views

Each camera in an overlap pair should be optimized for what it sees best:

```
Overlap pair: doorbell + entry_approach

Doorbell optimization:
  Primary signal: face (straight-on, close)
  Preprocessing: none or minimal (don't blur details)
  VLM focus: face recognition, micro-expression, greeting behavior
  Enrichment: heavy on face crops, pose

Entry approach optimization:
  Primary signal: gait + full body (side angle, distance)
  Preprocessing: edge-detection + slight contrast (emphasize body outline)
  VLM focus: re-ID embedding, gait analysis, posture, clothing
  Enrichment: heavy on full-body crops, skeleton keypoints

During overlap:
  Doorbell processes face → extracts face embedding
  Entry approach processes gait → extracts re-ID embedding
  Fusion: face + re-ID = high-confidence identity
  
  Optimization learning:
    "Doorbell preprocessing none helps face, edge-detect hurts" → keep minimal
    "Entry approach edge-detection helps gait, none is noisy" → keep edge-detect
```

## Behavioral biometrics across overlapping views

### Multi-signal behavioral patterns

Identity is strengthened by behavioral consistency across views:

```
Behavioral signals visible in overlapping cameras:

Greeting behavior (doorbell straight-on):
  - Eye contact pattern (does person look at camera?)
  - Smile (genuine vs. forced)
  - Hand gestures (wave, point, knock style)
  - Head tilt (attention to speaker)
  - Speech pattern (if audio: volume, pace, accent)

Body language (entry approach side angle):
  - Posture (confident, hesitant, aggressive)
  - Gait (bounce, stride length, toe-off, landing pattern)
  - Shoulder alignment (level or tilted)
  - Arm swing (symmetric or compensatory)
  - Walking speed under different contexts (rushed, leisurely, uncertain)

Dress code / clothing (full body visible in approach):
  - Outfit consistency (same person always wears similar style)
  - Seasonal appropriateness (winter coat in January, not July)
  - Accessory patterns (glasses, bags, watch always present)
  - Footwear pattern (always sneakers, never heels)

Social patterns (doorbell + interior):
  - Greeting familiarity (how long person stays at door vs. enters)
  - Interaction pattern with residents (waves to camera, looks for resident)
  - Duration of visit (quick vs. long stays)
  - Frequency pattern (weekly vs. monthly vs. occasional)

Example:
  Sarah's behavioral signature:
    Greeting: direct eye contact, genuine smile, quick wave
    Body: confident posture, 1.3 m/s pace (unhurried), even gait
    Clothing: typically jacket/sweater, comfortable jeans, sneakers
    Social: immediate entry (familiar), visits Thursday evenings
    Audio: cheerful greeting, distinct laugh pattern
  
  New detection: person matches 80% of these behavioral signals
    → Confidence boosted to 0.85 (behavioral consistency supports identity)
  
  Different detection: face 0.78 match but behavioral mismatch (wrong gait, 
                                                              hesitant body language)
    → Confidence reduced to 0.55 (face match contradicts behavior)
```

## Calibration benefits for identity (cross-reference §14)

When cameras in overlapping regions are calibrated:

### 1. Stereo triangulation enables height-based identity

```
Detection A (doorbell): person head at pixel (200, 300)
Detection B (entry_approach): same person head at pixel (450, 250)

Without calibration:
  Height estimate: unreliable, image-space based

With calibration:
  Stereo triangulation: person's head at (0.8m, -0.2m, 1.68m) world coords
  Height estimate: 1.68m ± 0.03m (very precise)
  
  Historical heights for candidates:
    Bob: 1.77m ± 0.02m (tall)
    Sarah: 1.62m ± 0.02m (shorter)
    Carlos: 1.75m ± 0.02m (tall)
  
  New person at 1.68m → best match is Sarah (0.06m diff, within tolerance)
  Likelihood: Sarah 0.89, Bob 0.15, Carlos 0.12
```

### 2. 3D trajectory verification

```
Calibrated world positions at sequential times:

10:15:00 — doorbell region: person at (0.5m, -0.1m, 0m) [ground level]
10:15:03 — overlap region: person at (0.8m, -0.2m, 0m)
10:15:06 — entry_approach: person at (1.5m, 0.5m, 0m)

Implied trajectory:
  Segment 1: (0.5, -0.1) → (0.8, -0.2) in 3s = 0.56m distance, 0.18 m/s speed
  Segment 2: (0.8, -0.2) → (1.5, 0.5) in 3s = 0.94m distance, 0.31 m/s speed
  
  Direction: consistent path from doorbell approach toward entry
  Speed: reasonable walking pace (varies due to pausing/looking)
  
  Anomaly check: is this trajectory physically possible?
    ✓ Yes — consistent direction, reasonable speed, no teleports
  
  Identity verification: if another person is elsewhere at same time:
    Bob at driveway (5.0m, 2.0m) at 10:15:04
    → Cannot be same person (5+ meters away, physically impossible to move)
    → Trajectory confirms different person
```

### 3. Temporal synchronization for stereo analysis

```
Stereo requires synchronized frames from both cameras.

With calibration, known:
  Camera A frame capture: exactly 10:15:32.145
  Camera B frame capture: exactly 10:15:32.146 (1ms offset, acceptable)
  
Stereo face verification:
  Both capture same person's face at same instant (1ms apart)
  → Can triangulate 3D face structure
  → Can verify biometric consistency (nose bridge position, eye distance)
  → Enables geometric face matching (not just appearance)
  
Without sync:
  Camera A: person's face at 10:15:32.0
  Camera B: person's face at 10:15:33.0 (1 second later)
  → Could be different person
  → Cannot do stereo triangulation
```

## Practical placement recommendations

### Minimum viable setup (high identity confidence)

```
Single floor house / apartment:

Must have:
  - Doorbell (straight-on): face recognition
  - Entry approach side (30–60° angle, 2–3m away): gait + full body
  - Interior hallway (if interior accessible): gait + posture consistency
  
Optional but recommended:
  - Driveway (if vehicle expected): vehicle + person identity
  - Garage/side entry: secondary entry point coverage
  
Calibration:
  - Doorbell ↔ entry approach MUST be calibrated (stereo baseline ~2m)
  - Hallway should be calibrated to doorbell (verify person entering home)
  
Result: high-confidence identity via overlapping views + multi-signal fusion
```

### Scaling: multi-level home

```
Ground floor:
  Front entry: doorbell (0°) + approach side (45°)
  Back entry: door camera + approach side
  Main hallway: 2+ angles (entry visible from both)
  Living room: 2 angles (multiple resident / guest scenarios)

First floor:
  Stairwell: top + bottom (stereo for fall detection)
  Hallway: continuous coverage from 2 angles
  Bedrooms: perimeter only (privacy)

Outdoor:
  Driveway: vehicle + pedestrian angles
  Backyard: perimeter cameras with overlap on high-traffic zones
  Pool: front + side (stereo + drowning detection)

Calibration:
  - All entry points calibrated to interior hallway (identity continuity)
  - Stairwell stereo calibration (fall detection)
  - Pool stereo calibration (life-safety)
  - Driveway to house (vehicle-to-person tracking)
```

## System requirements for multi-view fusion

1. **Overlap detection** — automatically identify which cameras have overlapping coverage
2. **Geometric registration** — from calibration (§14), compute 3D positions in world frame
3. **Temporal synchronization** — align frames from overlapping cameras (for stereo)
4. **Multi-modal fusion engine** — combine face + gait + behavioral + temporal signals
5. **Retroactive update mechanism** — when identity is confirmed, re-evaluate all past detections in overlapping regions
6. **Behavioral profile store** — per-person, per-camera behavioral signatures
7. **Anomaly detection** — identify implausible trajectories or identity conflicts
8. **Per-camera preprocessing tuning** — optimize what each camera extracts (face vs. gait vs. clothing)
